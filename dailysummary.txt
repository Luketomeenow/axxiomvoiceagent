1. REACH ANALYTICS ACCURACY FIX ("Who we reached")
=====================================================================
- PROBLEM (caught on the dashboard): the panel showed "61% reached a person"
  with Voicemail/IVR = 0. Audited against the data: of 218 "reached a person,"
  113 (52%) had voicemail/IVR/switchboard language in the transcript; only 16
  (7%) were a real back-and-forth. Voicemail/IVR read 0 only because detection
  was off and the new `ivr` disposition isn't deployed yet — so machines the
  agent monologued at were counted as people.
- ROOT CAUSE: "connected" = ended_by customer/agent, which can't tell a live
  human from a machine until answering-machine detection is on.
- FIX (honest labeling, no fabricated metric): headline -> "% answered"; green
  bar -> "Answered (incl. machines)"; an amber warning that it over-counts real
  people until detection is on. "Failed to place" relabeled as a dial error.

2. TWILIO TELEPHONY INTELLIGENCE -> ANALYTICS
=====================================================================
- AMD (human vs machine): switched Vapi's voicemailDetection provider from
  "vapi" to "twilio" (src/assistant/outbound/config.ts) so TWILIO runs machine
  detection and stamps the Call resource's AnsweredBy. twilioSync already reads
  that into call.answered_by -> a REAL human/machine/fax split. Bonus: same
  switch also gives the agent a machine signal (stop pitching machines).
- v_call_quality extended (scripts/sql/outbound_schema.sql): answered_human /
  answered_machine / answered_fax (AMD) + status_completed / busy / no_answer /
  failed / canceled (Twilio carrier disposition).
- Dashboard: "Who we reached" now shows the accurate AMD split ("X% reached a
  person," true machine %) WHEN AMD data is present, and falls back to the
  honest version otherwise. New "Carrier status (Twilio)" breakdown in Call
  quality. Everything degrades gracefully before the migration / before AMD.
- Twilio Conversation Intelligence: EVALUATED, SKIPPED. It duplicates Vapi's
  transcript/sentiment + our own Claude campaign-insights (objections /
  compliance / topics), targets human-agent call centers, and adds per-minute
  cost. Not worth it.
- Backend + web typecheck clean. Shipped on PR #5 (open).
